## ✅ Исправленные баги

### 1. Необъявленная переменная `user` в корзине ✅ ИСПРАВЛЕНО

**Файл:** `src/app/cart/page.tsx:59`

```typescript
if (!currentUserId && user)
```

**Проблема:** Переменная `user` используется в условии, но не объявлена в компоненте. Это вызовет `ReferenceError` при попытке оформить заказ.

**Почему это баг:** При оформлении заказа код попытается обратиться к необъявленной переменной, что приведёт к падению приложения.

**Исправление:** Необходимо либо удалить эту проверку, либо добавить правильную проверку `currentUserId`.

---

### 2. SSR проблема с window.matchMedia в ThemeContext

**Файл:** `src/context/ThemeContext.tsx:31`

**Проблема:** `window.matchMedia` вызывается без проверки на сервере, что может вызвать ошибку при серверном рендеринге.

**Почему это баг:** На сервере (или при SSG) объект `window` не существует, поэтому код падает.

**Исправление:** Добавить проверку:

```typescript
const mediaQuery = typeof window !== 'undefined' 
  ? window.matchMedia('(prefers-color-scheme: dark)') 
  : null;
```

---

### 3. Неправильная логика фильтра в fetchRestaurantMenu

**Файл:** `src/lib/supabase.ts:119-136`

**Проблема:** Функция делает первый запрос с неправильным фильтром:

```typescript
.eq('category_id', `category_${restaurantId}`)
```

Этот фильтр всегда возвращает пустой массив, так как формат `category_id` в базе данных другой. Затем делаются N+1 запросы для каждой категории последовательно.

**Почему это баг:** 
- Первый запрос всегда пустой
- Последовательные запросы вместо параллельных
- Неэффективная загрузка данных

**Исправление:** Исправить фильтр на правильный формат `category_id` и использовать `Promise.all` для параллельной загрузки.

**Рекомендация:** Исправить на подсчёт количества товаров:

```typescript
const itemCount = items.reduce((sum, item) => sum + item.quantity, 0);
{itemCount > 999 ? '999+' : itemCount}
```
